# 网页解析：UX 课程结构编辑器（截图）

## 1. 页面定位
- 页面类型：在线课程搭建后台（课程大纲/内容结构编辑）
- 课程标题：`Essentials of UX Design`
- 当前主流程步骤：`Structure`（第 4 步）
- 上游步骤：`Settings`、`Summary`、`Learning outcomes`
- 下游步骤：`Content`

## 2. 信息架构（IA）
- 顶部导航区：返回路径、语言切换、翻译按钮、多步骤进度导航。
- 左侧栏：`Learning outcomes`（学习目标）分层展示，包含 A/B/C 等目标及 A1/B1/C1 子目标。
- 中央主工作区：课程模块树（Module -> Section -> Lesson），支持层级折叠展开、拖拽排序、逐项编辑。
- 右侧浮层菜单（由 “+” 触发）：内容块类型库（Lesson/Video/Reading/Accordion/Flashcards/Quiz/Programming assignment/Peer Review 等）。

## 3. 核心交互能力
- 结构编辑：
- 支持多层级课程节点（如 1 -> 1.1 -> 1.1.1）。
- 每层节点有拖拽手柄和更多操作菜单（`⋮`）。
- 内容插入：
- 在节点位置通过 `+` 弹出组件库，按教学形态快速插入内容类型。
- 学习目标对齐：
- 每个内容节点右侧有 A1/B1/B2 等标签，表示内容与 Learning Outcomes 的映射关系。
- 全局批量动作：
- 左侧存在 `Regenerate All`，用于全量重生成内容。

## 4. 视觉与布局特征
- 布局模式：三栏工作台（左目标、中王结构、右弹层）。
- 视觉风格：浅灰中性色、卡片边界明确、文字层级清晰、偏工具型设计。
- 可读性：模块标题加粗，说明文案弱化，支持快速扫读。
- 反馈机制：当前步骤 `Structure` 高亮，下拉菜单覆盖层有滚动条提示可继续浏览更多组件。

## 5. 用户目标与使用场景推断
- 目标用户：课程设计者/教研人员/讲师。
- 主要任务：
- 从学习目标出发组织课程结构。
- 为不同节点补充多模态教学内容（视频、阅读、测验等）。
- 建立“目标-内容”可追踪对齐关系，便于课程质量验证。

## 6. 可落地页面需求（供后续前端生成）
- 必备模块：
- 顶部步骤导航条（含多步骤状态）。
- 左侧 Learning Outcomes 栏（目标卡片 + 子目标标签）。
- 中央树形结构编辑区（展开/收起、拖拽、节点菜单、节点说明文本）。
- 插件式内容选择弹层（带图标、名称、描述、滚动列表）。
- 必备交互：
- 节点新增、排序、折叠、删除/更多操作。
- 学习目标标签的选择与展示。
- 批量动作按钮（如 Regenerate All）。
- 响应式要求：
- Desktop（>=1200px）：三栏并行。
- Mobile（<=767px）：左栏与结构区需切换/抽屉化，弹层全屏化。
- Tablet（768-1199px）：左栏可折叠，结构区优先。

## 7. 风险与不确定项
- 截图仅覆盖单一状态，未展示：
- 拖拽过程态（占位、碰撞反馈）。
- 保存/发布/撤销机制。
- 错误态与空态。
- 未能确认是否支持多人协作、版本历史、权限分级。

## 8. 结论
该页面是一个以“学习目标驱动结构化教学设计”为核心的课程编排界面，重点在于：
- 结构树编辑效率
- 目标对齐可追踪性
- 多类型教学内容快速插入
